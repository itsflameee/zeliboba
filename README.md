Зелiбоба - bash-скрипт для удалённого завершения работы и перезагрузки "виндо-свинок" (Windows 10/11) с Linux-устройства (например Arch Linux или Debian, а также вытекающие из них дистро)

## Требования
Прежде чем запустить мой скрипт, убедись что ты выполнил все требования!

### Системные требования для Linux:
- Электричество (по возможности)

### Требования для установки скрипта
- Мозги (если повезёт)
- Мазохизм
- Linux (лучше Arch, однако если вы выполнили второй пункт, вероятнее всего у вас как раз Arch Linux)
- У вас нет друзей (иначе как бы вы скачали Linux)
- Вы не трогали траву за последние 10 лет
- Доступ в интернет (а то как ещё ты скачаешь моё никчёмное творение)

## Установка зелiбобы
Для того, чтобы установить зелiбобу вам необохдимо предварительно установить зависимости

### Обновите репозитории:
* **Arch-based:** *(Arch Linux, SteamOS, CachyOS, Manjaro)*
  
  ```
  sudo pacman -Sy
  ```
* **Debian-based:** *(Debian, Ubuntu, Mint)*
  
  ```
  sudo apt update
  ```
### Установите Git:

* **Arch-based:** *(Arch Linux, SteamOS, CachyOS, Manjaro)*
  
  ```
  sudo pacman -S git
  ```
* **Debian-based:** *(Debian, Ubuntu, Mint)*
  
  ```
  sudo apt install git
  ```

### Установите Samba:

* **Arch-based:** *(Arch Linux, SteamOS, CachyOS, Manjaro)*

  ```
  sudo pacman -S samba
  ```
* **Debian-based:** *(Debian, Ubuntu, Mint)*

  ```
  sudo apt install samba-common-bin
  ```
### Установка скрипта

Далее, вы можете начать установку самого скрипта

Рекомендуется устанавливать его в папку "/home/username/.local/bin/", так что в предложенных командах будет предствлена установка именно по этому пути, а также .desktop файл будет ссылаться именно туда.

```
mkdir -p ~/.local/bin # создаём директорию для бинарника, если ты ещё не засрал систему
cd ~/.local/bin # проваливаемся в директорию
git clone https://github.com/itsflameee/zeliboba.git # нагло качаем моё никому ненужное творение
cd zeliboba # проваливаемся в директорию моего никому ненужного творения
chmod +x pig_control.sh # даём права на запуск моего никчёмного bash скрипта
```

После установки, открой скрипт:

```
nano ~/.local/bin/zeliboba/pig_control.sh
```

(вероятно, nano не покажет тебе русских символов, так как это всё поделка НАТО, поэтому если ты на кедах, и уже засрал систему кедовскими приложениями, то пиши уже "kate ~/.local/bin/zeliboba/pig_control.sh")

И замени там всё что нужно (USER, PASS, MSG, они помечены комментариями, если тебе не лень - поизучай весь скрипт и измени под себя что надо). **Без изменений в USER и PASS скрипт не заработает вовсе!**

### Пропись в $PATH
Чтобы запускать зелiбобу простой командой ```zeliboba``` из любой точки системы, нужно прописать алиас и путь.
1. Если ты солидный человек, и сидишь на bash, пиши эти буковы:
```
nano ~/.bashrc
```

А если ты не менее солидный человек, но сидишь на zsh, пиши уже эти буковы:
```
nano ~/.zshrc
```

2. Пролистай в самый низ, и накалякай вот это вот:
```
# Чтобы терминал понимал команду зелiбобы
alias zeliboba='/home/USERNAME/.local/bin/zeliboba/pig_control.sh'

# Чтобы система видела папку зелiбобы
export PATH="$PATH:/home/USERNAME/.local/bin/zeliboba"
```

3. Ctrl + O, Enter, Ctrl + X
4. Если ты солидный человек, и сидишь на bash, пиши в этот раз эти буковы:
```
source ~/.bashrc
```

А если ты не менее солидный человек, но сидишь на zsh, пиши уже эти буковы:
```
source ~/.zshrc
```

### Создание .desktop

> [!IMPORTANT]
> Для создания .desktop файла, вам потребуется прописать в $PATH вашу зелiбобу!
> Сделайте это по гайду из прошлого шага
Если вы хотите запускать свою зелiбобу прямо с рабочего стола, или прочего места, способного умещать в себе ярлыки (как меню запуска приложений или панель задач в KDE) - создайте .desktop файлик. Ниже приведена инструкция:

Создайте где вам удобно .desktop файлик (например в ~/.local/share/applications/)

```
nano ~/.local/share/applications/zeliboba.desktop
```

И вставьте туда это содержимое:
```
[Desktop Entry]
Name=зелiбоба
Comment=Удалённый eboot виндо-свинок
Exec=konsole --hold -e "zeliboba"
Icon=/home/USERNAME/.local/bin/zeliboba/зелібоба.png
Terminal=false
Type=Application
Categories=Utility;System;
```

После этого вы можете пихать этот .desktop файлик хоть на рабочий стол, хоть себе в жопу, и вам никто и ничего не скажет!

### Настройка виндо-свинки
Без этого никуда. Microslop не дают нам достучаться до винды из под священного линукса просто так. Поэтому мы унижаем права виндо-свинок, и ставим свои правила.

Для начала открой PowerShell от имени администратора (наш аналог root в bash/zsh). Потом пиши туда вот эти буковки:

```
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v "LocalAccountTokenFilterPolicy" /t REG_DWORD /d 1 /f
```

Далее убедись что виндо-свинка запустила очередную мусорную службу, которая нам сегодня пригодится впервые за всё использование винды:
```Win + R``` > ```services.msc``` > Ищи "Remote Registry" или "Удалённый реестр" и ставь в свойствах тип запуска на "Автоматически", а потом запускай этот мусор.

И на всякий случай лучше отключи UAC (эта свинота может помешать), и пропиши в брандмауэр разрешение на порт 445.
